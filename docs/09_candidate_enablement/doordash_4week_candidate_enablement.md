# DoorDash-Style 4-Week Candidate Enablement Program

## Purpose
This program operationalizes a fixture + live-demo training system for intermediate data-analyst candidates preparing for marketplace growth case interviews.

Core outcomes:
1. Candidate can present a 5-slide, decision-ready narrative in 8 minutes.
2. Candidate can solve SQL Q1-Q4 patterns (including window functions).
3. Candidate can defend assumptions, trade-offs, and ROI logic under probing.

## Program Modes
- Fixture demo mode: Repeatable 48-hour take-home simulation with deterministic scoring.
- Live demo mode: Real-time interactive defense focused on prioritization, uncertainty handling, and executive communication.

## 4-Week Curriculum
| Week | Focus | Fixture Output | Live Output |
|---|---|---|---|
| 1 | Case decomposition + metric taxonomy + data QA | Issue tree, metric definitions, data-cleaning checklist | 15-min drill: define sales proxy and assumptions |
| 2 | Root-cause analytics + SQL fluency | Managed/unmanaged benchmark + funnel decomposition + timed SQL set | 25-min drill: explain joins and window logic out loud |
| 3 | Strategy + pilot + ROI trade-offs | Intervention shortlist, tiering model, control/treatment pilot, ROI model | 30-min drill: defend resource trade-offs with incomplete data |
| 4 | Executive storyline + pressure testing | Final 5-slide deck + full mock Q&A | 45-min live simulation + red-team interviewer challenge |

## Session Cadence
- Fixture sessions: 2 per week (30m framing, 45m analysis, 30m story, 15m QA).
- Live sessions: 2 per week (5m intake, 15m prioritization, 15m analysis sketch, 10m recommendation).
- Review session: 1 per week (rubric scoring, top failure patterns, targeted drills).

## Candidate Deliverable Contract
- Slide submission: max 5 slides (PDF).
- SQL submission: Q1-Q4 query file (PDF or doc).
- Analysis artifact: notebook/workbook with reproducible checks.
- Assumption log: explicit metric definitions, caveats, and unresolved unknowns.

## Acceptance Gates
1. Case completion gate
- Deadline <= 48 hours.
- Total score >= 75/100.
- No rubric section below 6/10 equivalent.
- SQL Q4 window-function logic is correct.

2. Live defense gate
- 8-minute presentation with 10 challenge questions.
- Candidate can revise recommendation without narrative contradictions.

3. Transferability gate
- Candidate repeats with non-DoorDash marketplace prompt.
- Framework reuse remains intact (decomposition, metric selection, trade-off logic).

## Operational KPIs
- Median score by rubric section.
- SQL accuracy by question type.
- Time-to-first-coherent storyline.
- Live-demo confidence trend.

## Iteration Rule
Run this with 3-5 candidates first. Rebalance week-level time allocation only if section-level average drops below 70.
